# Self-Hosting Guide

Run the entire exam-helper stack on your own server using Coolify and Docker Compose.

**What you'll have:**
- Backend API at `https://api.yourdomain.com`
- Web frontend at `https://yourdomain.com`
- Auth, email, and OAuth all wired up

---

## Prerequisites

| Requirement | Notes |
|---|---|
| A server running [Coolify](https://coolify.io) | Any VPS works — 1 vCPU / 1 GB RAM minimum |
| A domain you control | Two subdomains needed: root + `api.` |
| [MongoDB Atlas](https://cloud.mongodb.com) account | Free M0 tier is fine |
| [Google Cloud](https://console.cloud.google.com) project | For Google OAuth — free |
| [Resend](https://resend.com) account | For transactional email — free tier is 3 000 emails/month |

---

## Step 1 — MongoDB Atlas

1. Create a free **M0** cluster at [cloud.mongodb.com](https://cloud.mongodb.com)
2. Under **Database Access**, create a user with **Read and Write** permissions — note the username and password
3. Under **Network Access → Add IP Address**, add your server's IP (or `0.0.0.0/0` to allow all — fine for a personal instance)
4. Click **Connect → Drivers**, copy the connection string, replace `<password>` with your actual password:
   ```
   mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/?appName=Cluster0
   ```

---

## Step 2 — Google OAuth

1. Go to [console.cloud.google.com](https://console.cloud.google.com) → **APIs & Services → Credentials**
2. Click **Create Credentials → OAuth 2.0 Client ID**
3. Application type: **Web application**
4. Under **Authorized redirect URIs**, add:
   ```
   https://api.yourdomain.com/api/auth/callback/google
   ```
5. Save — copy the **Client ID** and **Client Secret**

---

## Step 3 — Resend (email)

1. Create an account at [resend.com](https://resend.com)
2. Go to **Domains → Add Domain** and verify your domain (add the DNS records shown)
3. Go to **API Keys → Create API Key** — copy it (starts with `re_`)
4. Your from-address will be something like `noreply@yourdomain.com`

---

## Step 4 — Deploy on Coolify

### 4a. Add your domain DNS records

In your DNS provider:

| Type | Name | Value |
|---|---|---|
| A | `@` (or `yourdomain.com`) | Your Coolify server IP |
| A | `api` | Your Coolify server IP |

### 4b. Create a new resource in Coolify

1. In Coolify → **New Resource → Docker Compose**
2. Connect your forked repo (or use the upstream `anasyd/exam-helper`)
3. Set **Compose file path** to `docker-compose.coolify.yml`

### 4c. Configure domains in Coolify

In the service settings, assign:
- `web` service → `https://yourdomain.com`
- `server` service → `https://api.yourdomain.com`

Coolify provisions SSL certificates automatically via Let's Encrypt.

### 4d. Set environment variables

In Coolify's **Environment Variables** panel, add only the variables you must supply — the others are auto-generated:

| Variable | Auto-generated? | Value |
|---|---|---|
| `BETTER_AUTH_SECRET` | **Yes** — Coolify fills this | — |
| `BETTER_AUTH_URL` | **Yes** — set from your domain | — |
| `FRONTEND_URL` | **Yes** — set from your domain | — |
| `MONGODB_URI` | No | Your Atlas connection string |
| `GOOGLE_CLIENT_ID` | No | From Step 2 |
| `GOOGLE_CLIENT_SECRET` | No | From Step 2 |
| `RESEND_API_KEY` | No | From Step 3 |
| `RESEND_FROM_EMAIL` | No | e.g. `noreply@yourdomain.com` |

> `BETTER_AUTH_SECRET` is auto-generated by Coolify using the `SERVICE_PASSWORD_64_BETTERAUTH` magic variable — a 64-character cryptographically random secret. It's generated once and stays stable across redeploys.

### 4e. Deploy

Click **Deploy**. Watch the logs — the `server` service starts first; `web` waits until the server healthcheck passes.

### 4f. Verify

```
https://api.yourdomain.com/api/health   → {"ok":true}
https://yourdomain.com                  → landing page loads
```

## Building from source (optional)

If you prefer to build the images yourself instead of pulling from GHCR, swap the `image:` lines in `docker-compose.coolify.yml` for `build:` directives:

```yaml
services:
  server:
    build:
      context: ./server
    # image: ghcr.io/anasyd/exam-helper-server:latest  ← remove this

  web:
    build:
      context: ./web
    # image: ghcr.io/anasyd/exam-helper-web:latest  ← remove this
```

## End-to-end smoke test

Run through this checklist after deployment:

- [ ] `https://api.yourdomain.com/api/health` returns `{"ok":true}`
- [ ] Sign up at `/sign-up` with email + password → verification email arrives
- [ ] Click the verification link → land on `/app` with banner dismissed
- [ ] Sign out → sign back in at `/sign-in`
- [ ] Click **Continue with Google** → Google consent → redirected to `/app`
- [ ] `/forgot-password` → reset email arrives → new password accepted
- [ ] `/app/profile` → update name → persisted on reload

## Troubleshooting

**Server healthcheck fails (Connection refused)**
→ Check Coolify logs for the `server` container — the most common cause is an invalid `MONGODB_URI` or the Atlas IP allowlist not including your server.

**`SSL routines: tlsv1 alert internal error` in server logs**
→ Your server's IP is not in the MongoDB Atlas Network Access allowlist. Add it under **Atlas → Network Access → Add IP Address**.

**Google OAuth returns `redirect_uri_mismatch`**
→ The redirect URI in Google Cloud Console must be exactly:
`https://api.yourdomain.com/api/auth/callback/google`

**Emails not arriving**
→ Check the Resend dashboard for delivery status. Make sure the domain is verified (green checkmark). Check spam.

**`NEXT_PUBLIC_AUTH_URL is not set` in browser console**
→ The `web` service's environment variable wasn't applied. Check Coolify's env panel and redeploy.
